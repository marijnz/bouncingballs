Game Objects

Each game object can have multiple components. A game object by itself has no capabilities other than positioning and orienting itself. In order to give a game object more capabilities, you can use components

Creating a Game Object

To create a game object in Lua, you need to use the global GameObjectManager and its function createGameObject. Every game object needs a globally unique ID => GUID. It can be used to get a game object instance from the GameObjectManager. This GUID is given when creating the game object.

local theGameObject = GameObjectManager:createGameObject("TheGuid")

In the example above, "TheGuid" is the GUID of the game object.

Components

Every component incorporates a certain capability, i.e. something that it can do. An example would be the RenderComponent which is capable of loading and rendering a given mesh. Below is a list of all currently available components and their functionality. But first we explain how to create a component in general.

Creating A Component

A game object serves as the factory for components. From Lua, you can create a component using a game object like this:

local theComponent = theGameObject:createXyzComponent(...)

As a concrete example, this is how you create a ScriptComponent:

local sc = theGameObject:createScriptComponent()
sc:setPath("data/models/TheNameOfTheModel.thModel")

The component is then created with the game object as the sole owner of that component. Components can not be shared between objects.

Component States

The following describes which states exist:

Value Can be Set by User
ComponentState.Initial no
ComponentState.Active yes
ComponentState.Inactive yes

By default, every component is set to ComponentState.Active after it has been initialized. Before that, it is in the state ComponentState.Initial.

To change the state of an individual component, you can call the function setState with either of the arguments given above which say "yes, can be set by user". If you want to change the state of all components that belong to a game object, you can call setComponentStats on that game object.

Render Component

Used to render a given mesh at the transformation of the owning game object. Use it like this:

local rc = theGameObject:createRenderComponent()
rc:setPath("data/models/theNameOfTheModelFile.thModel")
local thePath = rc:getPath() -- Returns the argument given above

Unlike any other object in the game, a render component supports scaling. You can manipulate the scale of the rendered object using setScale:

rc:setScale(rc:getScale():mulScalar(2.0)) -- Scale it to twice the original size
rc:setScale(Vec3(2.0, 1.0, 1.0)) -- Distorted in X direction

Physics Component

The physics component is used to create a rigid body which represents the game object in the physics world of the game. To create a rigid body, you have to use the construction info RigidBodyCInfo to define the properties of your rigid body:

local pc = theGameObject:createPhysicsComponent()
local cinfo = RigidBodyCInfo()
cinfo.motionType = MotionType.Fixed
cinfo.mass = 42
cinfo.shape = PhysicsFactory:createSphere(1.337)
cinfo.friction 0.33
-- ...
local rb = pc:createRigidBody(cinfo)

See the Physics page for more information.

Script Component

Used to update a certain game object. A script component has three 'slots': initialize, update, destroy. You can register any function with the script component for any of the given slots, provided your function has the correct signature.

Function Slot Signature
initialize function(gameObjectGuid)
destroy function(gameObjectGuid)
update function(gameObjectGuid, elapsedSeconds)

The syntax for registering those functions is simple:

function initialize(guid) print("Initializing " .. guid)
function destroy(guid) print("destroying " .. guid)
function update(guid) print("update " .. guid)

The most useful slot is the update slot. Since initialization happens only when There can only be exactly one function active for each of the slots

You can control whether the slots are called by setting the component state. See Component States for more details.
