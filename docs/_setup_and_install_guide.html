Setup and Installation Guide

Terminology

GPP = Gameplay Programming. It usually refers to the part of the engine/game code that is most relevant to the lecture you are attending.

GEP = Game Engine Programming. This is the engine used by GPP and the game framework and was created with the lecture of the same name.

Notes

  1. Throughout this guide, you will have to check, edit, or add new windows environment variables. You can either use the standard windows way and manage the environment variables in the system settings, or you can use the Rapid Environment Editor, which makes this a whole lot easier and more intuitive. There is a way to install a portable version, so you won't need administrator privileges.
  2. For those of you following this guide using one of the MI PCs, you will have to add your environment variables to your user variables instead of the system variables. Access the user variables via the control panel by opening the path Control Panel\User Accounts and Family Safety\User Accounts and clicking on "Change my environment variables". On german Windows, the path is Systemsteuerung\Benutzerkonten\Benutzerkonten where you have to click on Eigene Umgebungsvariablen Ã¤ndern.
  3. There is a mirror located at ftp://mirror.mi.hdm-stuttgart.de/Gaming/GEP_GPP/ where you will find all the files required to build and run the GEP/GPP engine/game in their correct versions. You have to access this page from within the HdM Network (e.g. via VPN). The folder structure works like this:
    • The folder Engine contains the engine and game sources itself.
    • The folder Setup contains all thirdparty requirements needed to build and run the game (e.g. DirectX SDK).
    • The folder Documentation contains the documentation for GPP.
    • The folder Tools contains stuff that is not explicitly required to run the game, but they make it easier for you somehow (e.g. the Havok Content Tools).
  4. If you need to set up your VPN, check the MI Wiki at https://wiki.mi.hdm-stuttgart.de/wiki/VPN.

Installation Guide

  1. Setting up Microsoft Visual Studio Professional 2012
    • Note: You absolutely cannot use any other compiler than Visual Studio 2012. What you can do is use another Visual Studio IDE to build and run GEP/GPP but you need the Visual Studio 2012 compiler. If you want to use GEP/GPP with Visual Studio 2013 for example, then do not "upgrage" the project files when opening the GEP.sln file, otherwise it will to use the VS2013 compiler instead of the VS2012 one.
    1. Download Microsoft Visual Studio Professional 2012 32-bit (English) from Dreamspark
    2. Once you purchased the software (don't worry, it's free for you), you can find the serial number on your online-receipt on the Dreamspark site.
    3. Mount the iso file of Microsoft Visual Studio Professional 2012 32-bit (English) as a DVD drive or burn it to a DVD and insert that one. Then run the .exe in the root directory (either called setup.exe, vs_professional.exe, or vs_ultimate.exe).
      • In case you are installing this on Windows 8, you can simply double-click the .iso file and Windows will automatically mount it.
      • In case you are NOT installing this on Windows 8, you can install WinCDEmu and use that to mount the .iso file.
    4. Install Visual Studio.
      • From the optional features you only need Microsoft Foundation Classes for C++.
      • The installation might take a while...
    5. Run Visual Studio once and close it again.
      • Only then will the installation be complete!
      • If it prompts you for default environment settings, we suggest you choose the C++ environment.
  2. Setting up the DirectX SDK
    1. Download the DirectX SDK and install it.
    2. Sometimes the installer will show an error message at the end of the installation process. This can usually be ignored.
    3. Confirm the DirectX installation.
      • Check the environment variable DXSDK_DIR, it has to point to the installation directory, e.g. C:\Program Files (x86)\Microsoft DirectX SDK (June 2010)\.
      • The following header files (.h) have to exist in C: Files (x86) DirectX SDK (June 2010)\\: D3D11.h, D3DX11.h
      • The following lib files (.lib) have to exist in C: Files (x86) DirectX SDK (June 2010)\\x64\: d3d11.lib, d3dx11.lib
  3. Setting up FMOD
    1. Download and install FMOD Studio.
      • NOTE: You specifically need version v1.02.10! Installing the newest version from the website will result in build error.
    2. Download and install FMOD Studio Programmer's API.
      • NOTE: You specifically need version v1.02.10! Installing the newest version from the website will result in build error.
    3. Create an environment variable FMOD_API that points to the installation directory of the FMOD Studio Programmer API.
      • By default, this would be C:\Program Files (x86)\FMOD SoundSystem\FMOD Studio API Windows
    4. To test the FMOD installation, run FMOD Studio and open the project exampled.fspro.
      • The playback of the sounds located in the "Events" tab should all work without problems. For example, select the sound Explosions\Single Explosion and click "Play" in the "Editor" tab.
  4. Setting up Havok
    1. Download and install the Havok SDK.
      • If you are running on one of the MI PCs, the havok SDK will already be installed there, so you can proceed with step 4.b.
      • Fill in your own Contact Information and the following for the Survey Information, before clicking on the "Submit" button:
        • Occupation: Student or Academic
        • Best Describing Intended Use: Educational Instruction/Course Study
        • Do you plan to: Other
      • On the following page, accept the Havok License Agreement (scroll all the way down...)
      • Now download the SDKs for Programmers according to your windows version
        • For Windows 7: SDKs for Programmers (2013.1.0, VS 2012 with SIMD, for Windows 7)
        • For Windows 8 or 8.1: SDKs for Programmers (2013.1.0, VS 2012 with SIMD, for Windows 8)
    2. Extract the archive somewhere where you can find it again.
      • It is not recommended to put it in C:\Program Files or C:\Program Files (x86) since it requires admin rights to write in these directories and it is possible that you will write files there later on, e.g. when compiling the havok demos.
    3. Create an environment variable HAVOK_API that points to the directory where you put havok.
      • On the MI PCs it points to C:\hk2013_1_0_r1\
  5. All done! If your Visual Studio was open during the whole setup process, you should restart it now so it will load the correct environment variables.

Troubleshooting

  • I am getting the error Failed to create DirectX 11 device when running the engine/game. Wtf?
    • You are probably running Windows 8.1. If so, install the Windows 8.1 SDK and everything should be fine. If you are running on another Windows version, try installing the according Windows SDK.
