1 Setup and Installation Guide {#SetupAndInstallGuide}
2 ============================

4 Terminology
5 -----------
6 
7 GPP = Gameplay Programming. It usually refers to the part of the engine/game code that is most relevant to the lecture you are attending.
8 
9 GEP = Game Engine Programming. This is the engine used by GPP and the game framework and was created with the lecture of the same name.

11 Notes
12 -----
13 
14 1. Throughout this guide, you will have to check, edit, or add new windows environment variables. You can either use the standard windows way and manage the environment variables in the system settings, or you can use the [Rapid Environment Editor][7], which makes this a whole lot easier and more intuitive. There is a way to install a portable version, so you won't need administrator privileges.
15 
16 2. For those of you following this guide using one of the MI PCs, you will have to add your environment variables to your user variables instead of the system variables. Access the user variables via the control panel by opening the path `Control Panel\User Accounts and Family Safety\User Accounts` and clicking on "Change my environment variables". On german Windows, the path is `Systemsteuerung\Benutzerkonten\Benutzerkonten` where you have to click on `Eigene Umgebungsvariablen Ã¤ndern`.
17 
18 3. There is a mirror located at ftp://mirror.mi.hdm-stuttgart.de/Gaming/GEP_GPP/ where you will find all the files required to build and run the GEP/GPP engine/game in their correct versions. You have to access this page from within the HdM Network (e.g. via VPN). The folder structure works like this:
19  - The folder __Engine__ contains the engine and game sources itself.
20  - The folder __Setup__ contains all thirdparty requirements needed to build and run the game (e.g. DirectX SDK).
21  - The folder __Documentation__ contains the documentation for GPP.
22  - The folder __Tools__ contains stuff that is not explicitly required to run the game, but they make it easier for you somehow (e.g. the Havok Content Tools).

24 4. If you need to set up your VPN, check the MI Wiki at https://wiki.mi.hdm-stuttgart.de/wiki/VPN.

26 Installation Guide
27 ------------------
28 
29 1. Setting up __Microsoft Visual Studio Professional 2012__
30  - _Note: You absolutely cannot use any other __compiler__ than Visual Studio 2012. What you __can__ do is use another Visual Studio __IDE__ to build and run GEP/GPP but you need the Visual Studio 2012 __compiler__. If you want to use GEP/GPP with Visual Studio 2013 for example, then do not "upgrage" the project files when opening the GEP.sln file, otherwise it will to use the VS2013 compiler instead of the VS2012 one._
31  1. Download `Microsoft Visual Studio Professional 2012 32-bit (English)` from [Dreamspark][1]
32  2. Once you _purchased_ the software (don't worry, it's free for you), you can find the serial number on your online-receipt on the [Dreamspark][1] site.
33  3. Mount the iso file of `Microsoft Visual Studio Professional 2012 32-bit (English)` as a DVD drive or burn it to a DVD and insert that one. Then run the .exe in the root directory (either called `setup.exe`, `vs_professional.exe`, or `vs_ultimate.exe`).
34  - In case you are installing this on Windows 8, you can simply double-click the .iso file and Windows will automatically mount it.
35  - In case you are NOT installing this on Windows 8, you can install [WinCDEmu][2] and use that to mount the .iso file.
36  4. Install Visual Studio.
37  - From the optional features you only need `Microsoft Foundation Classes for C++`.
38  - The installation might take a while...
39  5. Run Visual Studio once and close it again.
40  - Only then will the installation be complete!
41  - If it prompts you for default environment settings, we suggest you choose the C++ environment.
42 2. Setting up the __DirectX SDK__
43  1. Download the [DirectX SDK][3] and install it.
44  2. Sometimes the installer will show an error message at the end of the installation process. This can usually be ignored.
45  3. Confirm the DirectX installation.
46  - Check the environment variable `DXSDK_DIR`, it has to point to the installation directory, e.g. `C:\Program Files (x86)\Microsoft DirectX SDK (June 2010)\`.
47  - The following header files (`.h`) have to exist in `$(DXSDK_DIR)\Include\: D3D11.h, D3DX11.h`
48  - The following lib files (`.lib`) have to exist in `$(DXSDK_DIR)\Lib\x64\: d3d11.lib, d3dx11.lib`
49 3. Setting up __FMOD__
50  1. Download and install [FMOD Studio][4].
51  - NOTE: You specifically need version <span style="color:red">v1.02.10</span>! Installing the newest version from the website will result in build error.
52  2. Download and install [FMOD Studio Programmer's API][5].
53  - NOTE: You specifically need version <span style="color:red">v1.02.10</span>! Installing the newest version from the website will result in build error.
54  3. Create an environment variable `FMOD_API` that points to the installation directory of the FMOD Studio Programmer API.
55  - By default, this would be `C:\Program Files (x86)\FMOD SoundSystem\FMOD Studio API Windows`
56  4. To test the FMOD installation, run FMOD Studio and open the project `exampled.fspro`.
57  - The playback of the sounds located in the "Events" tab should all work without problems. For example, select the sound `Explosions\Single Explosion` and click "Play" in the "Editor" tab.
58 4. Setting up __Havok__
59  1. Download and install the [Havok SDK][6].
60  - _If you are running on one of the MI PCs, the havok SDK will already be installed there, so you can proceed with step 4.b._
61  - Fill in your own Contact Information and the following for the Survey Information, before clicking on the "Submit" button:
62  - Occupation: Student or Academic
63  - Best Describing Intended Use: Educational Instruction/Course Study
64  - Do you plan to: Other
65  - On the following page, accept the Havok License Agreement (scroll all the way down...)
66  - Now download the __SDKs for Programmers__ according to your windows version
67  - For Windows 7: `SDKs for Programmers (2013.1.0, VS 2012 with SIMD, for Windows 7)`
68  - For Windows 8 or 8.1: `SDKs for Programmers (2013.1.0, VS 2012 with SIMD, for Windows 8)`
69  2. Extract the archive somewhere where you can find it again.
70  - It is __not__ recommended to put it in `C:\Program Files` or `C:\Program Files (x86)` since it requires admin rights to write in these directories and it is possible that you will write files there later on, e.g. when compiling the havok demos.
71  3. Create an environment variable `HAVOK_API` that points to the directory where you put havok.
72  - On the MI PCs it points to `C:\hk2013_1_0_r1\`
73 5. __All done!__ If your Visual Studio was open during the whole setup process, you should restart it now so it will load the correct environment variables.

75 Troubleshooting
76 ----------------
77 
78 - I am getting the error __Failed to create DirectX 11 device__ when running the engine/game. Wtf?
79  - You are probably running Windows 8.1. If so, install the [Windows 8.1 SDK][8] and everything should be fine. If you are running on another Windows version, try installing the according Windows SDK.
