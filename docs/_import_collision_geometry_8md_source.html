2 ============================

4 General Notes {#ImportCollisionGeometry_Notes}
5 =============
6 
7 * In order to export your geometry as a havok collision shape, you have to use the Havok Content Tools which are only available for 3ds Max
8 * In 3ds Max, make sure that you use __Generic Units__ in "Customize -> Units Setup...", otherwise you will get weird scales.
9  * _Note: Do **not** use meters!_
10 
11 
12 Requirements{#ImportCollisionGeometry_Requirements}
13 ============
14 
15 Basically the same as in \ref ImportRenderGeometry_Requirements "requirements for importing render geometry" except that you will also need the _Havok Content Tools_. You can find them on the havok download page or in the [GEP/GPP Mirror][Mirror_Tools]. This implies that you __MUST use 3ds Max__ (or Maya, but we can't help you with that) since the havok content tools are only provided for 3ds Max (or Maya).

17 
18 Guide {#ImportCollisionGeometry_Guide}
19 =====
20 
21 We will start off where we ended in the \ref ImportRenderGeometry guide so we assume that you have a working terrain model in 3ds Max format (`terrain.max`) and a script where you tested it in (`terrain_test.lua`).
22 
23 With the Havok Content Tools installed, open `terrain.max` using 3ds Max. Select the terrain object in the scene and navigate to and click on the following option in the top-level menu
24 
25 ~~~
26 Havok Content Tools -> Physics -> Create Rigid Body
27 ~~~
28 
29 ![](ImportCollisionGeometry_CreateRigidBody.png)
30 
31 The havok content tools will now have added two new modifiers to the modifier list of your object, namely `Shape` and `Rigid Body`. Select the new `Shape` modifier and choose the correct `Shape Type`. For our terrain, we choose the shape type `Mesh`, because it is a non-convex (concave) object and cannot be accurately approximated using any of the standard primitives.
32 
33 ![](ImportCollisionGeometry_ShapeType.png)
34 
35 _Note: You could choose the `Rigid Body` modifier now and change some settings there as well, but GEP/GPP does not support loading of rigid bodies as such yet. All GEP/GPP can load for now is a havok shape, i.e. collision geometry_
36 
37 Next up, navigate to and click on
38 
39 ~~~
40 Havok Content Tools -> Export
41 ~~~
42 
43 ![](ImportCollisionGeometry_ExportButton.png)
44 
45 On most machines, this will take a while (~2 minutes) so go make yourself a coffee or a tea and wait until the havok opens up their export window. You could also browse http://soup.io or http://9gag.com, but that's probably going to consume a lot more than just 2 minutes...
46 
47 Once the Export window is open, you might see some warnings in the log window on the bottom but you can ignore them for now.
48 
49 _Note: In general, you should never just ignore a warning. It is better to fix the issue or disable the warning if possible. They are usually there for a reason._
50 
51 What we are going to do now is populating the "Configuration Set" (in the center at the top of the window) with some of the filters from the "Available Filters" section (top-left of the window). Keep in mind that the order, in which the filters are sorted in the "Configuration Set", is important.
52 
53 First, in the "Available Filters" section, go to the "Physics" tab. Select the filter "[Physics 2012] Create Rigid Bodies" and click on the "Add >>" button. On the right-hand side of the window, in the filter options, make sure the "Mesh Shape Type" is set to "BV compressed Mesh Shape" and ignore the other options.
54 
55 ![](ImportCollisionGeometry_PhysicsFilter.png)
56 
57 Next, in the "Available Filters" section, go to the "Core" tab. Select the filter "Write to Platform" and click on the "Add >>" button, just like before. On the right-hand side have to adjust some of the filter options.
58 
59 In the "Format" section, you have to make sure the "Binary" and the "Packfile" options are set. By default, "Binary" is already selected, but "Packfile" is not.
60 
61 The "Filename" is relative to the current .max file, so in our case we set to "`../data/collision/terrain.hkx`". You could also click on the "..." button and use the "Save File" prompt.
62 
63 The other options are not important for us.
64 
65 ![](ImportCollisionGeometry_WriteToPlatform.png)
66 
67 So far so good. The last thing to do for the actual export is to hit "Run configuration". But before you do, it is important that the filters in the "Configuration Set" are in the correct order, so make so make sure the physics filter comes first, then the core filter.
68 
69 Now hit the "Run Configuration" button and watch the export happen.
70 
71 ![](ImportCollisionGeometry_ExportLog.png)
72 
73 Alright! All that is left to do is use the collision from a Lua script. Check the next section on how to do this.

75 
76 Loading Collision Geometry from Lua
77 ===================================
78 
79 We will base our script on the one used in \ref ImportRenderGeometry_InGame of the render geometry import guide and make it look like this:
80 
81 ~~~{.lua}
82 include("defaults/physicsWorld.lua")
83 include("defaults/freeCamera.lua")
84 include("defaults/stateMachine.lua")
85 
86 local terrain = GameObjectManager:createGameObject("terrain")
87 terrain.renderComponent = terrain:createRenderComponent()
88 terrain.renderComponent:setPath("data/models/terrain.thModel")
89 
90 -- The part above is from the "importing render geometry" guide
91 
92 -- Make sure we will see the terrain
93 PhysicsSystem:setDebugDrawingEnabled(true)
94 
95 -- Create a physics component
96 terrain.physicsComponent = terrain:createPhysicsComponent()
97 
98 -- Create a rigid body using our externally created collision mesh
99 local cinfo = RigidBodyCInfo()
100 cinfo.motionType = MotionType.Fixed -- The terrain is fixed (static), of course
101 cinfo.shape = PhysicsFactory:loadCollisionMesh("data/collision/terrain.hkx")
102 terrain.rigidBody = terrain.physicsComponent:createRigidBody(cinfo)
103 ~~~
104 
105 Start the game now and you will see the wireframe of the collision mesh drawn on top of the terrain!
106 
107 Now to actually test if the collision geometry was set up correctly, we will create a dynamic rigid body and spawn it on top of the terrain. Since this is a dynamic body, it would be a good idea to have some forces acting so we actually see something happening in the physics world, so we create the world manually (instead of using the default) and set the gravity vector as well. The complete script then looks like this:
108 
109 ~~~
110 include("defaults/freeCamera.lua")
111 include("defaults/stateMachine.lua")
112 
113 do -- Physics world
114  local cinfo = WorldCInfo()
115  cinfo.gravity = Vec3(0, 0, -9.81)
116  physicsWorld = PhysicsFactory:createWorld(cinfo)
117  PhysicsSystem:setWorld(physicsWorld)
118 end
119 
120 -- Make sure we will see the terrain
121 PhysicsSystem:setDebugDrawingEnabled(true)
122 
123 local terrain = GameObjectManager:createGameObject("terrain")
124 terrain.renderComponent = terrain:createRenderComponent()
125 terrain.renderComponent:setPath("data/models/terrain.thModel")
126 
127 -- The part above is from the "importing render geometry" guide
128 
129 -- Create a physics component
130 terrain.physicsComponent = terrain:createPhysicsComponent()
131 
132 -- Create a rigid body using our externally created collision mesh
133 local cinfo = RigidBodyCInfo()
134 cinfo.motionType = MotionType.Fixed -- The terrain is fixed (static), of course
135 cinfo.shape = PhysicsFactory:loadCollisionMesh("data/collision/terrain.hkx")
136 terrain.rigidBody = terrain.physicsComponent:createRigidBody(cinfo)
137 
138 local ball = GameObjectManager:createGameObject("ball")
139 ball.physicsComponent = ball:createPhysicsComponent()
140 local cinfo = RigidBodyCInfo()
141 cinfo.motionType = MotionType.Dynamic
142 cinfo.shape = PhysicsFactory:createSphere(1.0) -- 1x1x1 meters
143 cinfo.mass = 1
144 cinfo.position = Vec3(0.0, 9.0, 3.0) -- Spawn the ball somewhere on the slope
145 ball.rigidBody = ball.physicsComponent:createRigidBody(cinfo)
146 ~~~
147 
148 When you start the game, you should see a rather big ball rolling up and down the half-pipe like terrain!
