Physics

For physics, GEP/GPP uses Havok Physics. Havok expects all distance units to be in meters.

The physics factory is used to create physics related objects. The most important type of such objects is the shape. Here's a list of the most commonly used shapes:

Shape Type Factory Call
ShapeType.Box createBox(Vec3(0.5, 0.5, 0.5)) -- 1x1x1 meters => These are half-extends!
ShapeType.Sphere createSphere(1.0) -- 1 meter radius
ShapeType.Capsule createCapsule(Vec3(0.0, 0.0, 0.0), Vec3(0.0, 0.0, 1.0), 1.0) -- start, end, radius => 1 meter height and radius
ShapeType.Cylinder createCylinder(Vec3(0.0, 0.0, 0.0), Vec3(0.0, 0.0, 1.0), 1.0) -- start, end, radius => 1 meter height and radius
ShapeType.Triangle createTriangle(Vec3(0.0, 0.0, 0.0), Vec3(1.0, 0.0, 0.0), Vec3(0.0, 0.0, 1.0)) => vertex0, vertex1, vertex2

A rigid body has many properties. Below you will find a list describing each of them with the name they can be found in the RigidBodyCInfo. The rigid body itself offers getters and setters to most of these properties, but not all of them. The shape for example can only be set in the RigidBodyCInfo that you pass to createRigidBody.

The most important ones are MotionType.Dynamic, MotionType.Fixed, and MotionType.Keyframed.

A dynamic rigid body is a fully simulated physics object that will get pushed around according to its mass and the force that is trying to push it.

A fixed rigid body, also commonly referred to as a static rigid body, is a physics object that has basically an infinite mass. It cannot be pushed around using forces and it is not simulated using linear velocity. You can use this as environmental geometry which should not move, such as a wall or the ground.

A key-framed rigid body is like a fixed rigid body except that it is movable using linear velocity. This is useful for all player controlled entities, such as the paddles in pong.
