10 Importing Model Data to the GEP Engine
11 --------------------------------------
12 
13 You will need
14 * The model converter from the [GEP/GPP Tools folder][GPP_Tools]
15 * A 3D modelling tool of your choosing that is able to export Collada (.DAE) or .FBX files, e.g.:
16  * 3ds Max 2013 (e.g. educational plan available for HdM students)
17  * [Blender 3D][Blender] Note: For __animated__ or physical __collision geometry__ you will still need 3ds Max or Maya!
18 
19 Acquire a model or create your own using 3ds Max 2013. Important: Make sure that the textures are in the .dds format (DXT5)! If necessary, convert them.
20 
21 Export the selected model from 3ds Max 2013:
22 ![](ExportSelected.png)
23 
24 Important: Select Z as the Up-Axis!
25 ![](ExportOptions.png)
26 
27 Use the `ModelConverterD.exe` to convert the `.dae` file (Collada) to the `.thModel` format. This can be done either on the command line or by using drag & drop.
28 ![](ModelConverterCommand.png)
29 
30 Additional Information:
31 * Copy the exported `.thModel` to `%%GEP%\data\models\`
32 * The texture paths in the model file need to be relative to the working directory of the game!
33  * E.g. The textures that the model `data/models/barbarian/barbarianStatic.thModel` uses are referenced in its .FBX file like this:
34  * `data/models/barbarian/Barbarian_Belt_Low_d.dds`
35  * `data/models/barbarian/Barbarian_Body_Low_d.dds`

45 ### Troubleshooting
46 
47 * All textures need to be quadratic and have to have a dimension of (2<SUP>N</SUP>)x(2<SUP>N</SUP>), e.g. 128x128, 256x256, etc.
48 * All textures need to be in `.dds` format using the __DXT5 compression__ setting.
49 * When converting your model to a `.thModel` using `ModelConverterD.exe`, make sure your model has UV coordinates, i.e. your model is unwrapped.
50  * Note: The model does not necessarily have to have a texture.
