1 State Machines {#StateMachine}
2 ==============
3 
4 Instantiation
5 =============
6 
7 There are basically two ways to create a state or a state machine: Use the low-level calls yourself, or use the functors `State`, `StateMachine`, and `StateTransitions`. Below you will see how to use the latter.
8 
9 Creating a State in an existing State Machine
10 ---------------------------------------------
11 
12 This can be used to hook into the main, built-in top-level state machine `/game`.
13 
14 ~~~{.py}
15 State{
16  -- The unqualified (local) name of the new state.
17  name = "myState",
18 
19  -- The fully qualified name of the parent state machine in which the state should be created
20  -- This is NOT optional here!
21  parent = "/game",
22 
23  -- This describes the listeners of the events of this new state.
24  -- This is optional and does not need to be specified.
25  eventListeners = {
26  enter = {
27  -- Here goes a list of anonymous functions
28  function(eventData) logMessage("Entered /game/myState") end,
29  function(eventData) logMessage("Yes, more than one listener is possible!") end
30  },
31  leave = {
32  function(eventData) logMessage("Left /game/myState") end
33  },
34  update = {
35  function(eventData) logMessage("Updating /game/myState") end
36  }
37  }
38 }
39 ~~~

41 Adding transitions to an existing State Machine
42 -----------------------------------------------
43 
44 The state transitions table must have a `"parent"` key but must be a pure array otherwise, i.e. it must only have consecutive integer keys (`t[1]`, `t[2]`, `t[3]`, ...)
45 
46 ~~~{.py}
47 StateTransitions{
48  -- The fully qualified name of the parent state machine. The states used in the transitions must exist in this state machine.
49  parent = "/game",
50 
51  -- Will always enter 'myState' when the '/game' state machine is entered.
52  -- The condition is optional, in which case a return value of true is assumed.
53  { from = "__enter", to = "myState" },
54 
55  -- When the ESCAPE key was pressed, the '/game' state machine will be left.
56  { from = "myState", to = "__leave", condition = function() return InputHandler:wasTriggered(Key.Escape) end },
57 
58  -- { ... } and so on.
59 }
60 ~~~

62 Creating a new State Machine
63 ----------------------------
64 
65 Below you will find two examples. The first shows how to create a new state machine that will be nested in a given parent. The second one shows how to create a top-level state machine (scroll down...)
66 
67 ~~~{.py}
68 -- Create a state machine "/game/nestedStateMachine"
69 StateMachine{
70  -- The unqualified (local) name for this new state machine.
71  -- If 'parent' is not specified, 'name' will become the qualified name
72  -- and this state machine will be created as a top-level state machine.
73  name = "nestedStateMachine",
74 
75  -- The fully qualified name of the parent state machine.
76  -- This is optional.
77  parent = "/game",
78 
79  -- Event listeners work the same with StateMachine as with State and are also optional.
80  eventListeners = { --[[ ... ]]-- },
81 
82  -- A list of inner states.
83  states = {
84  -- First inner state
85  {
86  -- Unqualified (local) name of the inner state.
87  name = "innerState",
88 
89  -- Note that you don't need to specify a parent here. If you do, you will get a warning.
90 
91  eventListeners = {
92  enter = { function(eventData) logMessage("Entered /game/nestedStateMachine/innerState") end }
93  }
94  },
95  -- Second inner state
96  {
97  -- Again, the unqualified (local) name of the inner state.
98  name = "anotherInnerState",
99 
100  -- Note now that the name is all that has to be specified for the simplest inner state.
101  },
102  -- And so on...
103  },
104 
105  -- A list of inner or rather nested state machines.
106  stateMachines = {
107  -- First nested state machine
108  {
109  -- This one is unqualified again
110  name = "innerMachine",
111 
112  -- No need to specify the parent here.
113 
114  -- Event listeners are optional...
115 
116  -- Here we go again...
117  states = {
118  {
119  name = "deepInside",
120  eventListeners = { enter = { function(eventData) logMessage("Entered /game/nestedStateMachine/innerMachine/deepInside") end } }
121  }
122  },
123 
124  transitions = { --[[ See the 'transitions' member below ]]-- }
125  },
126 
127  -- And so on...
128  },
129 
130  -- And finally, the transitions within this state machine.
131  -- Works just like using StateTransitions, except the 'parent' member does not need to be specified.
132  transitions = {
133  { from = "__enter", to = "innerState" },
134  { from = "innerState", to = "innerMachine" },
135  { from = "innerMachine", to = "__leave" },
136  }
137 }
138 ~~~
139 
140 Now we create a top-level state machine.
141 
142 ~~~{.py}
143 StateMachine{
144  name = "myTopLevel",
145  states = {
146  {
147  name = "level1",
148  eventListeners = {
149  update = {
150  function(eventData) logMessage("This will be shown many, many times!") end,
151  -- ...
152  }
153  }
154  }
155  },
156  transitions = {
157  { from = "__enter", to = "level1" }
158  -- Never leave level1 ...
159  }
160 }
161 ~~~
